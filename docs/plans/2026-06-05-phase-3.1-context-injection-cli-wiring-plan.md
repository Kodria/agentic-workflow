# Phase 3.1 — Context Injection CLI Wiring (W1 + W2) Implementation Plan
<!-- awm-qa-complete: 2026-06-05 -->

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cablear la capa de inyección de contexto (ya construida y testeada en Fase 3) al CLI, de modo que `awm doctor` reporte el estado de contexto de los agentes con injection y `awm init --agent opencode` realmente inyecte el contexto AWM en `opencode.json`.

**Architecture:** Dos puntos de integración. **W1** — `gatherMachine()` en diagnostics deja de devolver `contextInjection: []` (stub) y consulta `InjectionOrchestrator.contextStatus()` para cada agente con injection `config-instructions` cuyo config existe en disco (claude-code queda representado por `machine.hook`, sin duplicar). **W2** — un nuevo `stepContextInjection` en el orquestador de init llama `InjectionOrchestrator.installContext()` para el agente target cuando su injection no es `cc-settings-merge` (esa ruta la cubre `stepHook` por backward-compat) y aún no está `injected`. La I/O se inyecta vía `InitActions` para mantener el step puro y testeable.

**Tech Stack:** TypeScript/Node CLI, Jest (`jest --runInBand`). Cero dependencias nuevas. Reutiliza `InjectionOrchestrator`, `getInjection`, `PROVIDERS`, `REGISTRY_DIR`.

**Diseño de referencia:** `docs/plans/2026-06-04-multi-agent-decoupling-design.md` y la sección "Evaluación post-merge — Fase 3.1" de `docs/plans/2026-06-04-multi-agent-decoupling-plan.md`.

**Decisiones de diseño (locked):**
- **W1 sólo reporta agentes con injection `config-instructions` (hoy: opencode).** claude-code usa el hook como mecanismo de contexto, ya reportado como `machine.hook`; incluirlo en `contextInjection` duplicaría la fila. Por eso se filtra `inj.type !== 'config-instructions'`.
- **W1 reporta sólo si el config del agente existe en disco** (`fs.existsSync(inj.configPath)`). Así un usuario claude-only (sin `~/.config/opencode/opencode.json`) no recibe una fila `machine.context.opencode: missing` que degradaría el reporte sin motivo.
- **W2 decide skip/apply llamando a las acciones inyectadas** (`contextStatus`/`installContext`), no leyendo `ctx.machine.contextInjection`. Razón: los facts de W1 están *gated* por existencia de config; pero `awm init --agent opencode` debe inyectar aunque `opencode.json` no exista todavía (la estrategia lo crea). `inject()`/`materialize()` son idempotentes, así que es seguro.
- **`profileExtensions: []` (global baseline)** tanto en W1 (status) como en W2 (install), para que el hash materializado coincida con el esperado por status y el estado quede `injected` tras instalar. Contenido enriquecido por extensiones queda diferido (relacionado con W3, fuera de este plan).

**Diferido (no en este plan):** T1 spike manual de OpenCode (requiere opencode instalado, paso humano), W3 (`awm sync` regenera `~/.awm/context/awm-context.md`), scope `local`, `ConventionFileStrategy`.

// Estado del contexto AWM para agentes cuyo mecanismo de inyección es config-instructions
// (hoy: opencode). claude-code usa el hook → ya se reporta como machine.hook, no se duplica.
// Sólo se reporta un agente si su archivo de config existe (señal de que el agente está en uso).
function gatherContextInjection(): { agent: AgentTarget; state: InjectionState }[] {
    const out: { agent: AgentTarget; state: InjectionState }[] = [];
    const orch = new InjectionOrchestrator();
    for (const agent of Object.keys(PROVIDERS) as AgentTarget[]) {
        const inj = PROVIDERS[agent].injection;
        if (!inj || inj.type !== 'config-instructions') continue;
        if (!fs.existsSync(inj.configPath)) continue;
        let state: InjectionState = 'absent';
        try {
            state = orch.contextStatus({
                agent,
                scope: 'global',
                registryRoot: REGISTRY_DIR,
                installMethod: 'symlink',
                profileExtensions: [],
            });
        } catch {
            // registry ausente u otra falla → tratar como ausente (no romper el doctor)
            state = 'absent';
        }
        out.push({ agent, state });
    }
    return out;
}
```

/** Step 2b – Inject AWM context for agents whose mechanism isn't the Claude hook. */
export function stepContextInjection(d: InitDeps): StepResult {
    const inj = getInjection(d.agent);
    if (!inj) return ok('machine.contextInjection', 'machine', 'skipped', 'sin mecanismo de inyección');
    if (inj.type === 'cc-settings-merge') return ok('machine.contextInjection', 'machine', 'skipped', 'cubierto por hook');

    const op: ContextOp = {
        agent: d.agent,
        scope: 'global',
        registryRoot: d.registryRoot,
        installMethod: d.installMethod,
        profileExtensions: [],
    };
    if (d.actions.contextStatus(op) === 'injected') return ok('machine.contextInjection', 'machine', 'skipped');

    d.actions.installContext(op);
    return ok('machine.contextInjection', 'machine', 'applied');
}
```
