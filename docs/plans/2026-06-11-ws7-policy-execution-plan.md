> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ejecutar lo derivado de las 3 decisiones de política de WS-7 (registradas en `docs/plans/2026-06-09-distribution-roadmap.md`, sección WS-7): documentar Windows como no soportado (F-11), documentar mutation testing como opt-in (F-9), y normalizar a inglés todos los mensajes user-facing del CLI (F-10).

**Reglas del repo que aplican:**

## Platform support

AWM supports **macOS and Linux**. **Windows is not supported** at the moment: the
hook wrapper (`run-hook.cmd`) is a stub, skill installation relies on symlinks
(which require Developer Mode on Windows), and paths assume a Unix layout.

Native Windows support will be revisited if real demand appears.

### Task 2: F-9 — Documentar mutation testing como opt-in

**Files:**
- Modify: `docs/registry-guide.md` (nueva sección después de `## Publishing a new registry version`, línea ~103)

**Contexto para el implementador:** el sensor-pack js-ts (vive en `awm-baseline-registry`, NO en este repo) incluye un sensor `mutation` (Stryker) que `awm sensors init` escribe en `.awm/sensors.json` del proyecto con `enabled: false` (ver `cli/src/commands/sensors/init.ts:49`). El opt-in es por proyecto, editando ese archivo. No se toca el pack.

- [ ] **Step 1: Agregar sección al registry-guide**

Al final de `docs/registry-guide.md` (después de la sección `## Publishing a new registry version`):

```markdown
## Mutation testing (opt-in)

The js-ts sensor-pack ships a `mutation` sensor (Stryker) **disabled by default**
(`enabled: false`): mutation runs are slow and noisy as a per-commit gate, so AWM
treats them as an opt-in tool for critical paths rather than a default sensor.

To enable it in a project, edit the project's `.awm/sensors.json`:

```json
{
  "sensors": {
    "mutation": { "enabled": true }
  }
}
```

(only the `mutation` entry needs to change — keep the rest of the file as
generated by `awm sensors init`). Scope Stryker to the project's critical paths
via its own config (`stryker.conf.json`) to keep run times manageable. Run it
on demand with `awm sensors run --slow`.
```

- [ ] **Step 2: Verificar que el claim del doc es exacto**

Run: `grep -n "mutation" cli/src/commands/sensors/init.ts cli/src/commands/sensors/run.ts | head`
Expected: `init.ts` muestra `mutation:  { enabled: false }` y `run.ts` muestra que los sensores `--slow` incluyen mutation. Si `--slow` no incluye `mutation`, ajustar la última línea del doc al comando real.
