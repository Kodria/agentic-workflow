# Phase 3.2 — Context Regeneration on `awm update` (W3) Implementation Plan
<!-- awm-qa-complete: 2026-06-05 -->

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Hacer que `awm update` regenere el archivo de contexto global (`~/.awm/context/awm-context.md`) para los agentes cuya inyección quedó `stale`, cerrando el gap donde el registry se actualiza pero el contexto materializado queda desactualizado o ausente.

**Architecture:** Una función pura y testeable `regenerateGlobalContext()` itera los agentes con inyección `config-instructions`, consulta su `InjectionState` vía el `InjectionOrchestrator` existente, y para los que están `stale` (sentinel presente en la config pero el archivo materializado ausente o con hash viejo) re-ejecuta `installContext` para re-materializar el contenido canónico. El comando `awm update` la invoca tras `syncRegistry()`. No se toca a agentes `absent` (no se crean configs durante un update — eso es trabajo de `awm init`) ni `injected` (ya frescos).

**Tech Stack:** TypeScript/Node CLI, Jest (`jest --runInBand`), built-ins `fs`/`path` (cero dependencias nuevas). Reutiliza `InjectionOrchestrator`, `buildContext`, `materialize` ya construidos en Fase 3.

**Diseño de referencia:** `docs/plans/2026-06-04-multi-agent-decoupling-design.md` §6 (riesgo "Drift cross-máquina / sync"). Decisión de comando: **`awm update`** (no `awm sync`), confirmada con el usuario el 2026-06-05 — la staleness del archivo global nace tras el pull del registry, que es lo que hace `update`; `sync` es project-scoped.

**Decisiones de alcance:**
- **Solo scope global.** El archivo materializado es `~/.awm/context/awm-context.md`. El scope local de OpenCode sigue diferido (per §7 del diseño, shipping global-first).
- **`profileExtensions: []`** — consistente con el wiring existente (`stepContextInjection` en `init/steps.ts` y `gatherContextInjection` en `diagnostics/context.ts` ambos pasan `[]` para el contexto global).
- **Gate en `stale`.** `injected` → ya fresco, no se reescribe la config del usuario innecesariamente. `absent` (sentinel ausente) → no se auto-inyecta durante un update. Solo `stale` dispara regeneración.

## Contexto necesario para el implementador

El `InjectionOrchestrator` (en `cli/src/core/context/orchestrator.ts`) ya existe y expone:

```typescript
export type ContextOp = {
    agent: AgentTarget;
    scope: Scope;
    registryRoot: string;
    installMethod: 'symlink' | 'copy';
    profileExtensions: string[];
};

class InjectionOrchestrator {
    constructor(overrides?: { providerOverride?: ProviderConfig; contextPathOverride?: string });
    installContext(op: ContextOp): void;     // buildContext → materialize → strategy.inject (idempotente)
    uninstallContext(op: ContextOp): void;
    contextStatus(op: ContextOp): InjectionState;  // 'injected' | 'absent' | 'stale' — NO materializa
}
```

`contextStatus` para un agente `config-instructions` devuelve:
- `'absent'` — la config no existe, o existe pero `instructions[]` no contiene el sentinel (path del archivo materializado).
- `'stale'` — el sentinel está en `instructions[]` pero el archivo materializado está ausente, o su hash difiere del esperado.
- `'injected'` — sentinel presente y el archivo materializado coincide en hash.

`PROVIDERS` (en `cli/src/providers/index.ts`) se computa al cargar el módulo desde `process.env.HOME`. El `configPath` de opencode es `<HOME>/.config/opencode/opencode.json`. `REGISTRY_DIR` (en `cli/src/core/registry.ts`) es `<AWM_HOME>/cli-source`. `globalContextPath()` (en `materializer.ts`) es `<AWM_HOME>/context/awm-context.md`, evaluado en tiempo de llamada.

**Patrón de testing obligatorio** (igual que `cli/tests/core/diagnostics/context.test.ts`): setear `process.env.HOME` y `process.env.AWM_HOME` a un tmp dir, llamar `jest.resetModules()`, y luego `require()` el módulo fresco para que `PROVIDERS` y `REGISTRY_DIR` se recomputen con el HOME temporal.

```typescript
            JSON.stringify({ $schema: 'https://opencode.ai/config.json', instructions: entries }, null, 2));

        expect(() => regenerateGlobalContext()).not.toThrow();
        expect(regenerateGlobalContext()).toEqual([{ agent: 'opencode', action: 'skipped' }]);
```

```typescript
// cli/src/core/context/regenerate.ts
//
// W3 — Regeneración del contexto global tras `awm update`.
// Para cada agente con inyección config-instructions cuya config exista:
//   - 'stale'    (sentinel presente, archivo materializado ausente/viejo) → re-materializa.
//   - 'injected' (ya fresco)                                              → no toca nada.
//   - 'absent'   (sentinel ausente)                                       → no inyecta (eso es `awm init`).
// Solo scope global. Defensivo: nunca rompe `awm update` por una falla de un agente.

import fs from 'fs';
import { PROVIDERS, AgentTarget } from '../../providers';
import { REGISTRY_DIR } from '../registry';
import { InjectionOrchestrator, ContextOp } from './orchestrator';

export type RegenAction = 'refreshed' | 'fresh' | 'skipped';
export type RegenResult = { agent: AgentTarget; action: RegenAction };

export function regenerateGlobalContext(
    orch: InjectionOrchestrator = new InjectionOrchestrator(),
): RegenResult[] {
    const out: RegenResult[] = [];
    for (const agent of Object.keys(PROVIDERS) as AgentTarget[]) {
        const inj = PROVIDERS[agent].injection;
        if (!inj || inj.type !== 'config-instructions') continue;
        if (!fs.existsSync(inj.configPath)) continue; // agente no usado en esta máquina → omitir

        const op: ContextOp = {
            agent,
            scope: 'global',
            registryRoot: REGISTRY_DIR,
            installMethod: 'symlink',
            profileExtensions: [],
        };

        let state;
        try {
            state = orch.contextStatus(op);
        } catch {
            out.push({ agent, action: 'skipped' }); // registry u otra falla → no romper el update
            continue;
        }

        if (state === 'injected') { out.push({ agent, action: 'fresh' }); continue; }
        if (state === 'absent') { out.push({ agent, action: 'skipped' }); continue; }

        // state === 'stale' → regenerar el archivo materializado
        try {
            orch.installContext(op);
            out.push({ agent, action: 'refreshed' });
        } catch {
            out.push({ agent, action: 'skipped' }); // p.ej. using-awm ausente → no se puede regenerar
        }
    }
    return out;
}
```
