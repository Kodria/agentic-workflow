# Command Line Interface (CLI) Reference

The `awm` (Agentic Workflow Manager) binary is the entry point for interacting with the registry. It supports an interactive Text User Interface (TUI) via Clack Prompts by default, and a quiet non-interactive flag-based execution interface for automation via shell scripts or CI.

## Commands

### 1. `awm add [name]`

Install a specific Skill, Workflow, or bundled Process to your agent environment.

When run without arguments (e.g. `awm add`), it will launch an interactive search where you can visually select an artifact from your cached `.awm/registry/`.

#### Non-Interactive Flags

If you provide a valid `[name]` (e.g. `awm add documentacion`), you can bypass the interactive prompts by passing the required configuration via flags. This is highly recommended when scripting environment setups over `bash`.

| Flag | Required | Example | Description |
| ---- | -------- | ------- | ----------- |
| `-t, --type` | No | `--type process` | Explicitly state if you are installing a `skill`, `workflow`, or `process`. (AWM tries to auto-detect this if skipped). |
| `-a, --agent` | Yes | `--agent antigravity` | The destination IDE/Platform. Currently accepts `antigravity` or `opencode`. |
| `-s, --scope` | Yes | `--scope global` | The target installation. Currently accepts `global` (machine-wide) or `local` (inside your `pwd` repo's `.agents/` folder). |
| `-m, --method` | Yes | `--method symlink` | The installation hook. Either create a hard `symlink` to the AWM Cache, or eject via `copy`. |
| `-y, --yes` | No | `--yes` | Silently skips the final confirmation prompt (Yes/No) that occurs right before files are generated. |

**Example (Fully Scripted):**
```bash
# Add the cscti-docs-assistant skill globally via symlinks on antigravity without asking
awm add cscti-docs-assistant --type skill --agent antigravity --scope global --method symlink --yes
```

---

### 2. `awm list`

Outputs a formatted list to the terminal of all available artifacts found within the machine's Local Cache (`~/.awm/cli-source/registry/`).
This includes:
- Extracted Process bundles (from `processes.json`).
- Available single Skills.
- Available single Workflows.

*(No flags supported).*

---

### 3. `awm update`

Instantly pulls changes from the canonical Remote GitHub Registry into the machine's Local Cache.
Because AWM heavily utilizes `symlinks`, running `awm update` instantly patches all skills/workflows referencing that cache across every global and local installation on that machine.

*(No flags supported).*

---

### 4. `awm remove`

Interactively guides the user through uninstalling specific skills or workflows. It asks you which agent you want to modify, whether to target global or local scope, and then removes the detected symlink/folder.

*(Currently, `awm remove` does not support non-interactive execution flags).*

---

### 5. `awm --help`

Displays the classic `--help` text output generated by the underlying Commander package. Includes all versioning details.
